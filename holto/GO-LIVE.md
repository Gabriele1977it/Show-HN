# HOLTO — go-live checklist

A single punch list of everything that needs to be set or done, so nothing is
forgotten. Grouped by priority. Most items are **environment variables set on
Render** (`holto-api` service → Environment); a few are one-time drills.

## How to check status at a glance

You don't have to guess what's configured — the app tells you:

- **Admin → Integrations** — a green dot next to each key that's set.
- **Admin → Live data feeds** — freshness of the self-updating sources.

Set a value on Render → it redeploys → the dot turns green. Use that to tick this
list off.

---

## 1. Must-have — the app won't run without these

- [ ] **`DATABASE_URL`** — Postgres. *Auto-injected by the Render Blueprint.*
- [ ] **`SESSION_SECRET`** — signs logins. *Auto-generated by the Blueprint.*
- [ ] **`PUBLIC_URL`** / **`APP_ORIGIN`** — `https://holto-api.onrender.com` (for CORS + Stripe webhook).
- [ ] **`OWNER_EMAILS`** — your email, so you get the Admin panel + Pro tier.

## 2. Core experience — what makes HOLTO, HOLTO

- [ ] **`GEMINI_API_KEY`** — Ask HOLTO + document parsing. Free at aistudio.google.com.
- [ ] **`AIRLABS_API_KEY`** — live flight status.
- [ ] **`AERODATABOX_API_KEY`** — second flight source (catches pre-departure delays). ✅ *set via RapidAPI.*
- [ ] **`MAPBOX_TOKEN`** — geocoding, airport timing, best-light.
- [ ] **`GOOGLE_MAPS_API_KEY`** — Ask HOLTO nearby places (Places API enabled + key unrestricted for it).

## 3. Email

- [ ] **`RESEND_API_KEY`** + **`EMAIL_FROM`** — password resets & alerts. Free 3k/mo; sending domain verified.

## 4. Partners & monetisation

- [ ] **`AIRALO_CLIENT_ID`** / **`AIRALO_CLIENT_SECRET`** — eSIM sales. (Drop `AIRALO_BASE_URL` once you're off sandbox.)
- [ ] **`AWARDWALLET_API_KEY`** — loyalty import. ✅ *approved; key set; Render outbound IPs allow-listed.*
- [ ] **`STRIPE_SECRET_KEY`** / **`STRIPE_WEBHOOK_SECRET`** — Pro tier + eSIM checkout. Point the Stripe webhook at `…/api/stripe/webhook`.

## 5. Reliability & ops — the security-audit items

- [ ] **`SENTRY_DSN`** — error tracking. Create a free project at sentry.io, copy the DSN, add it. *(Dormant until set.)*
- [ ] **Restore drill** — run through [`RESTORE.md`](RESTORE.md) once (~15 min) to prove backups actually restore. Repeat quarterly.
- [ ] **Playwright baselines** — `cd holto/e2e && npm install && npx playwright install chromium && npm test -- --update-snapshots`, then commit the generated screenshots so the E2E workflow has references.
- [ ] **Branch protection** — GitHub → repo **Settings → Branches → Add rule** for your default branch → **Require status checks to pass** → select **HOLTO CI**. Now nothing merges unless typecheck + tests + build are green.

## 6. Optional tuning

- [ ] **`EXPO_ACCESS_TOKEN`** — higher push throughput (push works without it).
- [ ] `MONITOR_POLL_MS` / `FLIGHT_CACHE_TTL_MS` — leave at defaults unless quotas get tight.
- [ ] `OPENAI_API_KEY` — only if you want an AI fallback beyond Gemini (costs money; off by default).

---

## Launch-readiness summary

You're ready to invite real users once **sections 1–3** are all green in Admin →
Integrations (accounts, AI, flights, maps, email). Sections 4–5 make it earn
money and stay healthy; section 6 is polish. Everything not yet green is a
value you paste into Render — no code changes needed.
